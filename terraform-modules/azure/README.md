# terraform-modules/azure

Reusable Terraform modules for Azure infrastructure. Each module is independently versioned and can be composed via Terragrunt.

## Module Index

| Module | Description |
|--------|-------------|
| [vnet/](vnet/README.md) | Azure Virtual Network, subnets (with delegation + service endpoints) |
| [aks/](aks/README.md) | AKS cluster with system + user node pools, Workload Identity, Key Vault CSI |
| [mysql/](mysql/README.md) | Azure MySQL Flexible Server with private DNS, VNet delegation |
| [aks-addons/](aks-addons/README.md) | AKS addon sub-modules (metrics-server, ingress-nginx, cluster-autoscaler, Key Vault CSI, Workload Identity) |

## Design Principles

- **Provider generated by Terragrunt** — no `provider.tf` in module directories; credentials are injected at deploy time via Terragrunt's `generate` block
- **Workload Identity** — Azure's OIDC-based identity mechanism (equivalent to AWS EKS Pod Identity); no service principal secrets stored in state
- **Modular addons** — each addon is a separate Terraform module and separate Terragrunt state, mirroring the AWS `eks-addons` pattern
- **Terraform >= 1.10.0** with native state locking

## Provider Version

```hcl
azurerm = "~> 4.0"
helm    = "~> 2.17"  # (addon modules only)
```

## Usage

Modules are consumed by Terragrunt configs in `terragrunt/azure/`. See each module's README for input/output details.

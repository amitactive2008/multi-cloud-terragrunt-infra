# terraform-modules/azure/mysql

Creates an Azure MySQL Flexible Server with private VNet integration, private DNS zone, and auto-generated administrator credentials. Databases are created as separate resources so they can be managed individually.

## Resources Created

| Resource | Description |
|----------|-------------|
| `azurerm_resource_group` | Resource group for the MySQL server |
| `random_password` | Auto-generated administrator password |
| `azurerm_private_dns_zone` | Private DNS zone for the server FQDN |
| `azurerm_private_dns_zone_virtual_network_link` | Links DNS zone to the VNet |
| `azurerm_mysql_flexible_server` | MySQL Flexible Server |
| `azurerm_mysql_flexible_database` (×N) | One per entry in `var.databases` |

## Usage

```hcl
module "mysql" {
  source              = "../../terraform-modules/azure/mysql"
  name                = "dev-mysql"
  resource_group_name = "dev-data-rg"
  location            = "eastus"

  mysql_version  = "8.0.21"
  sku_name       = "B_Standard_B1ms"
  storage_size_gb = 20

  administrator_login      = "mysqladmin"
  database_subnet_id       = module.vnet.database_subnet_id
  private_dns_zone_vnet_id = module.vnet.vnet_id

  databases = ["appdb"]

  tags = { Environment = "dev", ManagedBy = "terraform" }
}
```

## Inputs

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `name` | `string` | yes | MySQL Flexible Server name |
| `resource_group_name` | `string` | yes | Resource group (created by this module) |
| `location` | `string` | yes | Azure region |
| `mysql_version` | `string` | no | MySQL version (default: `8.0.21`) |
| `sku_name` | `string` | no | SKU (default: `B_Standard_B1ms`) |
| `storage_size_gb` | `number` | no | Storage in GB (default: `20`) |
| `iops` | `number` | no | Storage IOPS (default: `396`) |
| `administrator_login` | `string` | yes | Admin username |
| `database_subnet_id` | `string` | yes | Delegated subnet for MySQL VNet integration |
| `private_dns_zone_vnet_id` | `string` | yes | VNet ID to link to the private DNS zone |
| `databases` | `list(string)` | no | Database names to create |
| `backup_retention_days` | `number` | no | Default: `7` |
| `geo_redundant_backup_enabled` | `bool` | no | Default: `false` |
| `tags` | `map(string)` | no | Tags applied to all resources |

## Outputs

| Name | Description |
|------|-------------|
| `server_name` | MySQL Flexible Server name |
| `server_fqdn` | Fully-qualified domain name |
| `resource_group_name` | Resource group name |
| `administrator_login` | Admin username |
| `administrator_password` | Admin password (sensitive) |
| `database_names` | List of created database names |

## Notes

- The server is deployed with **VNet delegation** — it is only reachable from within the VNet (no public endpoint)
- The administrator password is generated by `random_password` and stored in Terraform state. Consider storing it in Azure Key Vault after first deploy
- The subnet must be pre-delegated to `Microsoft.DBforMySQL/flexibleServers`

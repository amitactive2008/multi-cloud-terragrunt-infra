# AWS RDS Module

Creates an Amazon RDS MySQL instance with automated password management via AWS Secrets Manager, a dedicated security group, and a DB subnet group.

## Architecture

```
RDS Module
├── Security Group            — restricts port 3306 to specified security groups
├── DB Subnet Group           — uses the DB subnets from the VPC module
├── RDS Instance (MySQL 8.0)  — Multi-AZ optional, storage autoscaling, backup
├── Secrets Manager Secret    — stores host, port, dbname, username, password
└── Secret rotation           — credentials auto-rotated (optional)
```

## Usage

```hcl
module "rds" {
  source = "../../terraform-modules/aws/rds"

  name   = "prod-mysql"
  vpc_id = module.vpc.vpc_id

  db_subnet_ids = module.vpc.db_subnet_ids

  # Allow EKS nodes to connect
  allowed_security_group_ids = [module.eks.node_security_group_id]

  db_name        = "appdb"
  master_username = "admin"
  instance_class  = "db.t3.micro"

  allocated_storage     = 20
  max_allocated_storage = 100

  multi_az            = false
  deletion_protection = false
  skip_final_snapshot = true

  tags = {
    Environment = "prod"
  }
}
```

## Inputs

| Name | Description | Type | Default | Required |
|---|---|---|---|---|
| `name` | Name prefix for all RDS resources | `string` | — | yes |
| `vpc_id` | VPC ID | `string` | — | yes |
| `db_subnet_ids` | Subnet IDs for the DB subnet group | `list(string)` | — | yes |
| `allowed_security_group_ids` | Security groups allowed on port 3306 | `list(string)` | `[]` | no |
| `db_name` | Initial database name | `string` | `"appdb"` | no |
| `master_username` | Master DB username | `string` | `"admin"` | no |
| `instance_class` | RDS instance class | `string` | `"db.t3.micro"` | no |
| `allocated_storage` | Initial storage in GB | `number` | `20` | no |
| `max_allocated_storage` | Storage autoscaling upper limit in GB | `number` | `100` | no |
| `multi_az` | Enable Multi-AZ deployment | `bool` | `false` | no |
| `deletion_protection` | Enable deletion protection | `bool` | `false` | no |
| `skip_final_snapshot` | Skip final snapshot on deletion | `bool` | `true` | no |
| `backup_retention_period` | Days to retain automated backups | `number` | `7` | no |
| `tags` | Additional tags | `map(string)` | `{}` | no |

## Outputs

| Name | Description |
|---|---|
| `db_instance_id` | RDS instance identifier |
| `db_instance_arn` | RDS instance ARN |
| `endpoint` | Hostname of the RDS instance |
| `port` | Port of the RDS instance |
| `db_name` | Database name |
| `username` | Master username |
| `secret_arn` | ARN of the Secrets Manager secret |
| `secret_name` | Name of the Secrets Manager secret |

## Credentials

The master password is generated by Terraform and stored in AWS Secrets Manager as a JSON object:

```json
{
  "username": "admin",
  "password": "<generated>",
  "host":     "prod-mysql.<id>.<region>.rds.amazonaws.com",
  "port":     3306,
  "dbname":   "appdb"
}
```

The secret ARN is passed to the `eks-addon` module so pods can mount credentials via the Secrets Store CSI driver.
